Urban Rivals API

Urban Rivals API

Introduction to the Urban Rivals API

You are a developing a website, a flash animation, a dekstop application or even a mobile application and you want to access Urban Rivals's public datas like (characters details, market history) or player's datas (battle log, friends feed, collection...) to create useful tools for our players.

Urban Rivals is providing an API for this exact purpose. As our API system is authenticated by the OAuth protocol, you will need to get a consumer key (and secret) before accessing our remote methods. Think of OAuth as a secure tunnel to call methods on our system on behalf of an user without asking for his credentials at any time. The user will be asked to allow of refuse access on his behalf for your application and will be able to revoke access anytime if he wants to.

Explaining how to use OAuth in your specific language is beyond the scope of this documentation, however, you can find a lot of documentations on the official website: oauth.net as well as various ready-made library for your programming language of choice there: oauth.net/code. Googling about OAuth will also return a lot of results :)

Once you get a consumer key and secret for the Urban Rivals API, you can start working on the next incredible application for Urban Rivals players.

OAuth specs

  • Request a token: https://www.urban-rivals.com/api/auth/request_token.php
  • Authorize a token: https://www.urban-rivals.com/api/auth/authorize.php
  • Convert an authorized token into a access token: http://www.urban-rivals.com/api/auth/access_token.php
  • Call remote methods using access token: https://www.urban-rivals.com/api/
  • Signing methods: HMAC-SHA1 or RSA-SHA1 (PLAINTEXT is not supported)
  • Https Errors & API logic:
    • 401: unable to valid oauth signature (wrong token or wrong consumer key)
    • 404: non valid call (check your syntax)
    • 405: consumer doesnt have access to one of the call

Calling remote methods

Once you have setup your OAuth consumer and have done all the necessary steps to get your access token, you can call remote methods on our API.

To make a request to our api, send an OAuth signed request using either GET/POST/PUT with one http parameter: request.

The request parameter must be encoded in json (only - for now) and be an array of calls, each call beeing an associative array, below is an example of a request doing 2 calls with the first call having 1 param and the second having 2 params.

As you have understood by now, you can make multiples calls in one request, and you should do it as much as you can. It limits the http round-trip and allow your application to feel more responsive to the user. Of course it's not always possible, especially if you need the response of a previous call to make the next one.

Receiving responses

Since it's based on HTTP, all requests are done synchroneously meaning you'll get the response immediatly.

The response will be an array of associative array, one for each call you made in your request, encoded in json (only - for now). Each associative array will be populated with a sub assoc array named "context" with some contextual datas about the response (ie: like the page you asked for, the total pages...) and may have another sub assoc array named "items" with each item's details (ie: if you asked for a collection page, you'll get an items array for each collection card in the page with characters name, character id etc...).

Don't be afraid, it's really simple to use and pretty powerful.

Reducing the response size

You can also filter the response directly on the server to avoid data traffic. Along with the call & it's params, you can optionnally add "contextFilter" and "itemsFilter" to get only the values you are interrested in. The syntaxe is simple: for example, if you wish to get only the player's id and name from the context returned by general.getPlayer, pass this as the contextFilter along with your call: array('player.id', 'player.name'). Of course, you might need to adapt it to your own programming language.

  • imageSize

characters.getCharacters Requires Public access to the API, returns Items & Context

Get the list of availables characters, with an optional list of characterIDs or an optional clanID filter and sorting options.

orderby can be 'asc' or 'desc'
Set the maxLevels params to true if you want the fully levelled character.

Parameters:
  • charactersIDs
  • clanID
  • sortby
  • orderby
  • imageSize
  • imageFormat
  • maxLevels
